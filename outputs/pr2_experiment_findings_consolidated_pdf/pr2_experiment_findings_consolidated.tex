\section*{Consolidated Findings}
\begin{itemize}
  \item Decode horizon is critical for label validity; short horizons can collapse positives (e.g., \texttt{max\_tokens=2048} for \texttt{k=20}).
  \item ROC-AUC should be retained but never alone under imbalance; PR-AUC and positive-class metrics are required.
  \item Earlier ID--OOD deltas were confounded by sparse positives; matched-label datasets are required for fair view comparisons.
  \item Final-layer views are close on ROC-AUC; \texttt{last\_token\_final} is more stable on macro-F1 while \texttt{mean\_pool\_final} can lift PR-AUC.
  \item In the k=5 three-view study, posterior (completion) features yield strong macro-F1 ($\sim$0.80), while prefill views sit around $\sim$0.67.
  \item OOD performance remains the bottleneck under current labels; gains likely need data/label improvements before larger models.
\end{itemize}

\section*{Current PR \#2 Codebase State}
\begin{itemize}
  \item Shared multi-view dataset build so one rollout-label pass can feed multiple feature views.
  \item Feature-key aware training/evaluation paths for selecting a specific view from shared datasets.
  \item Imbalance-aware metrics added to evaluation logging (PR-AUC, positive-class precision/recall/F1, prevalence).
  \item Official RFM runner integration and result artifacts.
  \item New k=5 three-view feature modes and mixed-stage (prefill + completion) extraction.
  \item MLP architecture controls (hidden dim, depth, dropout) plus k=5 dataset/ablation SLURM launchers.
  \item Rollout data-parallel IPC hardened to avoid semaphore rebuild failures.
  \item Runtime hardening: canonicalized cache guard for HOME, prefill-only builds skip retaining rollout token IDs, and explicit \texttt{--probe-preset} is required when a checkpoint lacks \texttt{probe\_config}.
  \item Aggregation now reselects from \texttt{metrics.jsonl} whenever selection/tie-breaker differ or cannot be inferred, preventing stale default summaries.
  \item RFM training now falls back to a best-row selection when all ranking metrics are NaN.
  \item Completion-view prompt+rollout merges are now built per batch to reduce host RAM pressure.
\end{itemize}

\section*{Recommended Next Step}
Now that the k=5 three-view ablation is complete, decide whether to invest in stronger prefill modeling (richer aggregators or larger MLPs) or scale the multi-source dataset rebuild under the same fixed label protocol and metrics.
